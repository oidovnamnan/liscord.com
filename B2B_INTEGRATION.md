# 🔗 LISCORD — БИЗНЕС ХООРОНДЫН ИНТЕГРАЦИ (B2B Marketplace)

> **Зарчим:** Liscord дээрх бизнесүүд өөр хоорондоо холбогдож, үйлчилгээ солилцож чадна.
> Онлайн дэлгүүр → Карго-д ачаа захиалж, → Хүргэлтийн компанид хүргэлт дамжуулна.
> Бүгд нэг платформ дотор, нэг товшилтоор.

---

## 1. 🌐 ЕРӨНХИЙ ОЙЛГОЛТ

### 1.1 Бизнес хоорондын холбоосын загвар

```
┌──────────────┐     Ачаа захиалга     ┌──────────────┐
│  📱 Онлайн   │ ──────────────────── → │  📦 Карго     │
│  Дэлгүүр     │                        │  Компани      │
│ (Захиалагч)  │     Бараа ирлээ        │ (Үйлчилгээ)  │
│              │ ← ──────────────────── │              │
└──────┬───────┘                        └──────────────┘
       │
       │  Хүргэлт хүсэлт
       ▼
┌──────────────┐
│  🚚 Хүргэлт  │
│  Компани      │
│ (Үйлчилгээ)  │
└──────────────┘

Мөн:
┌──────────────┐     Тавилга захиалга   ┌──────────────┐
│  🏢 Оффис    │ ──────────────────── → │  🪑 Тавилга   │
│  Компани      │                        │  Үйлдвэр     │
└──────────────┘                        └──────────────┘

┌──────────────┐     Нэрийн хуудас      ┌──────────────┐
│  Ямар ч      │ ──────────────────── → │  🖨️ Хэвлэх    │
│  Бизнес      │                        │  Газар        │
└──────────────┘                        └──────────────┘

┌──────────────┐     Бөөнөөр авах       ┌──────────────┐
│  📱 Жижиглэн │ ──────────────────── → │  🏪 Бөөний    │
│  Дэлгүүр     │                        │  Нийлүүлэгч   │
└──────────────┘                        └──────────────┘
```

### 1.2 Гол ойлголтууд

| Ойлголт | Тайлбар |
|---------|---------|
| **Үйлчилгээ үзүүлэгч (Provider)** | Карго, хүргэлт, хэвлэл, бөөний — үйлчилгээ санал болгодог бизнес |
| **Захиалагч (Consumer)** | Тухайн үйлчилгээг ашигладаг бизнес |
| **Үйлчилгээний холбоос (Link)** | 2 бизнес хоорондын тохиролцсон холбоо |
| **Хүсэлт (Service Request)** | Нэг бизнесээс нөгөө бизнес рүү илгээсэн захиалга |
| **Хуваалцсан мэдээлэл** | 2 бизнесийн хооронд автомат дамждаг мэдээлэл |

## 2. 🔗 ИНТЕГРАЦИЙН ТӨРЛҮҮД

### 2.1 📦 Карго интеграци (Онлайн дэлгүүр → Карго компани)

**Хэрэглээ:** Онлайн дэлгүүр Хятадаас бараа авчирдаг. Карго компаниар ачаа тээвэрлүүлдэг.

#### Flow:
```
1. Онлайн дэлгүүр: Шинэ захиалга ирсэн (iPhone 15 Pro x2)
2. Бараа нь Хятадад байгаа → "Карго захиалга илгээх" товчлуур дарна
3. Холбогдсон карго компанид автомат захиалга үүснэ:
   - Илгээгч: Хятад дахь нийлүүлэгч
   - Хүлээн авагч: Онлайн дэлгүүр (эсвэл шууд эцсийн харилцагч)
   - Бараа: iPhone 15 Pro x2
   - Жин: 0.4кг
   - Тайлбар: "Онлайн дэлгүүрийн #ORD-0042 захиалга"
4. Карго компани бараа хүлээн авч, статус шинэчилнэ
5. Статус бүр онлайн дэлгүүрт автомат дамжина:
   - "Агуулахад ирсэн" → Онлайн дэлгүүрийн #ORD-0042 "Бараа ирж байна" 
   - "Монголд ирсэн" → #ORD-0042 "Бараа ирлээ, бэлтгэж байна"
```

#### Хуваалцах мэдээлэл:
| Consumer → Provider | Provider → Consumer |
|---------------------|---------------------|
| Барааны нэр, тоо | Tracking дугаар |
| Хүлээн авагчийн нэр, утас | Статус шинэчлэлт |
| Жин (ойролцоогоор) | Бодит жин |
| Тайлбар | Тээврийн төлбөр |
| Яаралтай эсэх | Хүлээгдэж буй ирэх огноо |
| | Бараа ирсэн мэдэгдэл |

---

### 2.2 🚚 Хүргэлтийн интеграци (Ямар ч бизнес → Хүргэлт компани)

**Хэрэглээ:** Дэлгүүр, хоолны газар, цэцгийн газар → Хүргэлтийн үйлчилгээнд хүргэлт дамжуулна.

#### Flow:
```
1. Онлайн дэлгүүр: Захиалга бэлэн болсон → "Хүргэлт захиалах" товчлуур
2. Хүргэлтийн компанид хүсэлт автомат үүснэ:
   - Авах хаяг: Дэлгүүрийн хаяг
   - Хүргэх хаяг: Харилцагчийн хаяг
   - Бараа: 1 боодол, ~0.5кг
   - Холбоо барих: Харилцагчийн утас
   - Тусгай зааварчилгаа: "Эмзэг бараа"
3. Хүргэлтийн компани хүлээн авч, жолооч хуваарилна
4. Статус дамжина:
   - "Жолооч хуваарилагдсан" + жолоочийн нэр, утас
   - "Авсан" (дэлгүүрээс авсан)
   - "Хүргэж байна"
   - "Хүргэгдсэн" → Онлайн дэлгүүрийн захиалга автомат "Хүргэгдсэн"
```

#### Хүргэлтийн хүсэлтийн мэдээлэл:
| Талбар | Тайлбар |
|--------|---------|
| `pickupAddress` | Авах хаяг (Provider бизнесийн хаяг) |
| `pickupContact` | Авах үеийн холбогдох хүн, утас |
| `deliveryAddress` | Хүргэх хаяг |
| `deliveryContact` | Хүлээн авагчийн нэр, утас |
| `deliveryDistrict` | Дүүрэг |
| `packageSize` | Жижиг / Дунд / Том / Маш том |
| `packageWeight` | Жин (ойролцоо) |
| `isFragile` | Эмзэг эсэх |
| `isUrgent` | Яаралтай эсэх |
| `scheduledTime` | Хэзээ хүргэх (ASAP / тодорхой цаг) |
| `specialInstructions` | Тусгай зааварчилгаа |
| `estimatedFee` | Хүргэлтийн төлбөр (автомат тооцоо) |
| `sourceOrderId` | Эх захиалгын дугаар |
| `sourceOrderNumber` | #ORD-0042 |

### 2.3 🏪 Нийлүүлэлтийн интеграци (Жижиглэн дэлгүүр → Бөөний нийлүүлэгч)

**Хэрэглээ:** Жижиглэн дэлгүүр → Бөөний нийлүүлэгчээс бараа нөхцлөөр авна.

#### Flow:
```
1. Жижиглэн дэлгүүр: Нөөц дуусч байна → "Нийлүүлэгчээс захиалах"
2. Бөөний бизнесийн каталогоос бараа, тоо сонгоно
3. Бөөний бизнест захиалга автомат үүснэ:
   - Бараа жагсаалт + тоо ширхэг
   - Үнэ: Бөөний шатлал үнээр автомат
   - Төлбөрийн нөхцөл: Тохиролцсон (зээл / шууд)
4. Бөөний бизнес бараа бэлтгэж, хүргэж / хүлээлгэж өгнө
5. Жижиглэн дэлгүүрийн нөөц автомат нэмэгдэнэ
```

#### Хуваалцах мэдээлэл:
| Consumer → Provider | Provider → Consumer |
|---------------------|---------------------|
| Барааны жагсаалт + тоо | Үнийн санал (бөөний) |
| Хүргэлтийн хаяг | Захиалгын баталгаа |
| Хүссэн огноо | Бэлэн болох огноо |
| | Нэхэмжлэл |
| | Хүргэлт мэдэгдэл |
| | → Нөөц автомат нэмэх |

---

### 2.4 🖨️ Хэвлэлийн интеграци (Ямар ч бизнес → Хэвлэх газар)

**Хэрэглээ:** Нэрийн хуудас, стикер, сав баглаа, банер хэвлүүлэх.

---

### 2.5 🍔 Хоол + Хүргэлт интеграци

**Хэрэглээ:** Хоолны газар → Хүргэлтийн үйлчилгээ (2.2-тэй адил, гэхдээ хоолны онцлогтой).

#### Нэмэлт мэдээлэл:
- Хоол халуун байх шаардлагатай → Яаралтай хүргэлт
- Бэлтгэлийн хугацаа → Жолоочид мэдэгдэх цаг
- Олон захиалга → Нэг жолооч олон хүргэлт (batch)

## 7. ⚙️ ТОХИРГОО & ЭРХ

### 7.1 Интеграцийн тохиргоо

```
/app/settings/integrations/
├── Үйлчилгээний профайл (Provider)
│   ├── Санал болгох үйлчилгээ on/off
│   ├── Үнэ тохируулах
│   ├── Үйл ажиллагааны цаг
│   ├── Хамрах хүрээ
│   └── Нийтэд харагдах эсэх
│
├── Холбоосууд (Consumer)
│   ├── Холбогдсон бизнесүүд
│   ├── Автомат зөвшөөрөх on/off
│   ├── Мэдэгдлийн тохиргоо
│   └── Төлбөрийн горим
│
└── Автомат дүрмүүд
    ├── "Захиалга бэлэн → автомат хүргэлт хүсэлт"
    └── "Нөөц дуусвал → автомат нийлүүлэгчээс захиалах"
```

### 7.2 Шинэ эрхүүд
