## 1. 🎯 АСУУДЛЫН ТОДОРХОЙЛОЛТ

### 1.1 Өнөөгийн байдал
Монголын жижиг, дунд бизнесүүд (ялангуяа карго, бөөний худалдаа, хоол хүргэлт, гар утасны дэлгүүр гэх мэт) өдөр бүр олон тооны захиалга хүлээн авдаг. Эдгээр захиалгыг ихэнх нь **Google Sheets** эсвэл **цаасан дээр** бүртгэдэг.

### 1.2 Google Sheets-ийн хязгаарлалтууд
| # | Асуудал | Нөлөө |
|---|---------|--------|
| 1 | **Гар аргаар оруулалт** | Алдаа гардаг, удаан |
| 2 | **Хайлт, шүүлт хязгаарлагдмал** | 1000+ мөр болоход олоход хэцүү |
| 3 | **Статус хянах боломжгүй** | Захиалга хаана байгаа нь тодорхойгүй |
| 4 | **Олон хүн зэрэг ажиллахад** | Мэдээлэл давхцах, устгагдах |
| 5 | **Мобайлаас ашиглахад муу** | Гар утсан дээр sheets нээхэд тохиромжгүй |
| 6 | **Тайлан, нэгтгэл** | Борлуулалтын тайлан гаргахад маш хэцүү |
| 7 | **Мэдэгдэл байхгүй** | Шинэ захиалга ирэхэд мэдэхгүй |
| 8 | **Түүх хадгалдаггүй** | Хэн юу өөрчилсөн нь тодорхойгүй |
| 9 | **Харилцагчийн бүртгэл** | Давтагдсан харилцагчдыг хянах боломжгүй |
| 10 | **Аюулгүй байдал** | Хэн ч мэдээлэл устгаж, өөрчилж чадна |

### 1.3 Зорилтот хэрэглэгчид

**Анхдагч зорилтот бүлэг:**
- 📦 Карго, импортын бизнесүүд (Хятад→Монгол бараа)
- 🛍️ Бөөний болон жижиглэн худалдаа
- 🍔 Хоол хүргэлтийн газрууд
- 📱 Гар утас, электроник дэлгүүрүүд
- 👗 Хувцас, гоо сайхны бизнес (Instagram/Facebook дэлгүүрүүд)

**Нийтлэг шинж чанар:**
- Өдөрт 10-500+ захиалга авдаг
- 1-20 ажилтантай
- Одоо Google Sheets, Excel, Messenger-ээр захиалга бүртгэдэг
- Техникийн мэдлэг дундаж

### 3.2 Яагаад Firebase?
| Давуу тал | Тайлбар |
|-----------|---------|
| **Сервергүй** | Backend код бичих шаардлагагүй, хурдан хөгжүүлэлт |
| **Бодит цаг** | Firestore snapshot listener → мэдээлэл шууд шинэчлэгдэнэ |
| **Офлайн** | Firestore offline persistence → интернэтгүй ч ажиллана |
| **Масштабтай** | 10 хэрэглэгчээс 10,000 хүртэл автомат хэмжигдэнэ |
| **Аюулгүй** | Security Rules + Auth → роль-д суурилсан хандалт |
| **Туршлагатай** | Таны өмнөх төслүүдэд ашиглагдсан |

### 5.2 Хэрэглэгчийн гол аяллууд (User Flows)

#### Аялал 1: Шинэ захиалга бүртгэх (Хамгийн чухал flow)
```
1. Ажилтан /app/orders руу орно
2. "➕ Шинэ захиалга" товчлуур дарна
3. Харилцагч сонгох/үүсгэх:
   ├── Утасны дугаар бичнэ → автомат хайлт
   ├── Өмнөх харилцагч олдвол → сонгоно
   └── Шинэ бол → нэр, утас оруулна (2 талбар л хангалттай)
4. Бараа нэмэх:
   ├── Барааны нэр бичнэ → автомат хайлт каталогоос
   ├── Олдвол → тоо ширхэг оруулна, үнэ автомат
   └── Каталогт байхгүй бол → нэр, үнэ гараар оруулна
5. Нэмэлт мэдээлэл (optional):
   ├── Хүргэлтийн хаяг
   ├── Тэмдэглэл
   └── Төлбөрийн мэдээлэл
6. "Хадгалах" → Захиалга үүснэ, #ORD-0042 дугаар автомат олгогдоно
```

#### Аялал 2: Захиалгын статус шинэчлэх
```
1. Захиалгын жагсаалтаас захиалга дээр дарна
2. Статус товчлуур дарна → Дараагийн статус руу шилжинэ
   (Жишээ: "Шинэ" → "Баталсан" → "Бэлтгэж байна" → "Хүргэлтэнд" → "Хүргэгдсэн")
3. Эсвэл swipe хийж статус солино (мобайл дээр)
4. Статус өөрчлөгдөхөд → түүхэнд бүртгэгдэнэ (хэн, хэзээ)
```

#### Аялал 3: Захиалга хайх, шүүх
```
1. Хайлтын талбарт:
   ├── Захиалгын дугаар: "ORD-0042"
   ├── Харилцагчийн нэр: "Болд"
   └── Утасны дугаар: "8800 1234"
2. Шүүлтүүр:
   ├── Статусаар (олон сонголттой)
   ├── Огнооны хүрээгээр
   ├── Төлбөрийн статусаар
   └── Үүсгэсэн ажилтнаар
3. Эрэмбэлэх: Огноо, дүн, статус
```
